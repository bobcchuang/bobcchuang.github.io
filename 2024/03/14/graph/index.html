Graph 分享

基本介紹

  • Graph，是由 Nodes與 Edges組成的一種資料結構。
  • Graph DB是用來儲存圖形結構資料的資料庫，屬於一種非關聯式資料庫(NoSQL)
  • 目的：透過樹狀結構、網狀結構的方式，將各節點之間的關係儲存，使複雜的資料關係容易被理解和查詢。

Graph 基本元素

中 英 範例(SNA)
節點/頂點 node / vertex / point 人、車、理賠案件
關聯/邊/弧/連結 edge / connection / relationship 開(車)、付款、被保
屬性 attribute / property 報案日期、理賠金額
標籤 label / type 人、車、理賠案件

  • node與edge可以有各自的attribute與label

Graph 種類

  • 依邊的種類分：無向圖 vs 有向圖

  • 依是否含平行邊或環分：簡單圖 vs 多重圖

  • 依上述關聯，Graph共可分成4種:

    • 無向簡單圖
    • 無向重圖
    • 有向簡單圖
    • 有向重圖

  • SNA選用”無向重圖“

    • 後續應用中，關聯不需要區分方向
    • 節點間有多重的關聯，”人投保車”、”人開車”

Graph 常用指標

  • 中心性(Centrality): 衡量圖形中的節點重要性程度。
    • 度數中心性（Degree Centrality）：
      • 計算該節點相鄰的關聯數量。
      • 你的社會關係越多，你的影響力越強。
    • 介數中心性（Betweenness Centrality）：
      • 計算有多少最短路徑經過該節點。
      • 處在越多其他成員的最短路徑上，越是核心成員。
    • 接近中心性（Closeness Centrality）：
      • 計算該節點到其他所有節點的最短路徑長度的平均。
      • 比介數中心性更接近幾何上的中心位置。
    • 特徵向量中心性（Eigenvector Centrality）：
      • 計算相鄰節點的degree總和。
      • 跟你有關的人越重要，你也越重要。
  • 最短路徑分析（Shortest Path）：用於找到兩個節點之間最短的路徑或路線。用於交通、電信的路線規劃; 或是社交網路中，人與人的關係遠近。
  • 社群偵測（Community Detection）： 找出圖形中相關或密切連接的節點子集，這些子集被視為社群或群體。

應用場景

理賠詐欺偵測

紀錄保險中，人、車與案件的關聯，從而檢測可能的詐領行為，提高保險公司對風險的感知能力。

推薦引擎

將客戶興趣、朋友和購買歷史紀錄等各類資訊間的關係存放為圖形的形式。根據有相似興趣及購買歷史記錄的其他人所購買的產品，向使用者推薦產品。

資料血緣

紀錄資料庫中表格的關聯，當資料表發生異常時，可由資料血緣關係中，找出異常的源頭，以及後續需要校正的表格。

道路規劃和物流優化

用於分析城市道路、交通流量和物流路徑，幫助最佳化路徑規劃和資源分配。

補充

樹

如果一個無向簡單圖 G 滿足以下相互等價的條件之一，那麼 G 是一棵樹：

  • G 是沒有環的連通圖。
  • G 沒有環，但是在 G 內添加任意一條邊，就會形成一個環。
  • G 連通，但是如果去掉任意一條邊，就不再連通。
  • G 中任意兩頂點間存在唯一路徑。

走遍節點的演算法(DFS & BFS)

  • 深度優先搜尋DFS

    • 每當走到一個節點，就會以那個節點為新起始點，往其中一邊搜尋到底或下一個節點。當已經走遍節點其中一邊的所有可能，才會開始走節點的另外一邊。

    • （一條路走到底，沒路再回上一個路口走其他路）

    • （類似於迷宮摸牆走的作法）

  • 廣度優先搜尋BFS

    • 先選定一個節點開始走訪，逐一走過此節點相鄰未被走過的節點，若相鄰節點都被走過，再從走訪過的節點中擇一為起點，繼續走訪，以此類推。

    • （走一步退一步，沒路就到新路口繼續走一步退一步）

  • 在簡單圖形中兩者差異不大，在複雜圖形中

    • 尋找最短路徑： BFS 優於 DFS (時間少)
    • 列舉所有路徑： DFS 優於 BFS (空間少)

儲存Graph的兩種方式

  • Adjacency Matrix

    • 使用較多空間
    • 走遍所有edges慢
    • 找特定edges快

  • Adjacency List

    • 使用較少空間

    • 走遍所有edges快

    • 找特定edge慢

  • 現實世界中，Graph 資料通常是更傾向於散落的。此時 Adjacency List 優於 Adjacency Matrix。
